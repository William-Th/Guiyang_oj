### REV101 - 提交草稿进行审核

**功能描述**: 验证能将草稿题目提交给审核人审核

**测试步骤**:
1. 使用`createDraftQuestion`辅助函数创建草稿题目（数学-七年级）并获取题目编码（questionCode）
2. 进入草稿箱 tab
3. 使用题目编码精确定位目标行（filter hasText questionCode），点击该行的"发布"按钮
4. 在发布对话框中选择第一个可用审核人
5. 勾选"练习"适用范围
6. 点击提交

**验证点**:
1. 发布对话框正确显示，标题为"发布题目"
2. 审核人下拉列表正确加载
3. 适用范围checkbox可正常勾选
4. 提交后显示成功消息

**优先级**: P0
**状态**: ✅ 通过
**备注**: 使用题目编码精确定位提升测试准确性 | 需检查checkbox状态避免重复点击

**更新日期**: 2025-01-20

### REV103 - 审核题目-批准通过

**功能描述**: 验证双老师审核流程 - teacher01 创建并提交，teacher02 审核批准

**测试步骤**:
1. **teacher01 创建草稿**:
   - 使用 `loginAsTeacher` 以 teacher01 身份登录
   - 使用时间戳创建唯一题目内容：`【REV103-${timestamp}】测试审核批准功能 - 2+2=4`
   - 调用 `createDraftQuestion` 创建判断题（数学-七年级-L1）
2. **teacher01 提交审核**:
   - 使用 `navigateToQuestionBank` 点击导航进入题库页面
   - 点击"我的草稿" tab
   - 使用时间戳内容精确定位题目行：`.filter({ hasText: \`【REV103-${timestamp}】\` })`
   - 点击"发布"按钮，打开发布弹窗
   - 选择审核人为 teacher02 (王老师)
   - 勾选"练习题库"范围（检查 checkbox 状态避免重复点击）
   - 点击"提交"按钮
3. **teacher02 审核批准**:
   - 使用 `loginAsTeacher` 切换到 teacher02 身份
   - 导航到题库页面，点击"待我审核" tab
   - 使用时间戳内容查找待审核题目
   - 点击"审核"按钮，打开审核弹窗
   - 选择"批准通过"
   - 填写审核意见："题目质量良好，内容准确，批准通过。"
   - 点击"提交审核"按钮

**验证点**:
1. teacher01 提交审核后显示成功消息，模态框关闭
2. teacher02 能在"待我审核"中看到待审核题目
3. 审核弹窗正确显示，标题"审核题目"
4. 批准操作成功后显示成功消息

**优先级**: P0
**状态**: ✅ 通过
**备注**: 使用时间戳数据隔离 + 点击导航策略 + 双老师流程，测试稳定通过 (22.1s)

**更新日期**: 2025-10-21

### REV106 - 多选适用范围提交审核

**功能描述**: 验证提交审核时能选择多个适用范围（练习题库 + 测评题库）

**测试步骤**:
1. **teacher01 创建草稿**:
   - 使用 `loginAsTeacher` 以 teacher01 身份登录
   - 使用时间戳创建唯一题目内容：`【REV106-${timestamp}】测试多选范围功能 - 光合作用`
   - 调用 `createDraftQuestion` 创建判断题（生物-七年级-L1）
2. **teacher01 提交审核并选择多个范围**:
   - 使用 `navigateToQuestionBank` 点击导航进入题库页面
   - 点击"我的草稿" tab
   - 使用时间戳内容精确定位题目行
   - 点击"发布"按钮，打开发布弹窗
   - 选择审核人为 teacher02 (王老师)
   - 勾选"练习题库" checkbox（检查状态避免重复点击）
   - 勾选"测评题库" checkbox（检查状态避免重复点击）
   - 点击"提交审核并发布"按钮（注意按钮文本与其他测试不同）

**验证点**:
1. 能够同时勾选多个适用范围 checkbox
2. 所有勾选的 checkbox 状态正确（使用 `evaluate` 检查原生 DOM 状态）
3. 提交成功并显示成功消息
4. 模态框关闭

**技术要点**:
- 使用 `checkbox.evaluate((el: HTMLInputElement) => el.checked)` 检查 checkbox 状态
- 只在未勾选时点击，避免取消勾选
- 等待 UI 稳定后再点击提交按钮

**权限要求**:
- teacher02 必须同时拥有 `question_bank_review` 和 `assessment_review` 权限
- "练习题库" 对应 `question_bank_review` 权限
- "测评题库" 对应 `assessment_review` 权限

**优先级**: P1
**状态**: ✅ 通过
**备注**: 权限配置问题已修复，测试稳定通过 (17.9s)

**问题诊断**: 初始失败是因为 teacher01/teacher02 缺少 `assessment_review` 权限，已通过数据库迁移文件修复

**更新日期**: 2025-10-21
