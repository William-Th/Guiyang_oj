# 贵阳小学生测评服务平台 MVP 规划

## 1. 项目概述

### 1.1 目标用户
- **主要用户**: 贵阳市小学生（6-12岁）
- **次要用户**: 教师、家长、教育局管理人员
- **客户**: 贵阳市教育局/教育部门

### 1.2 核心价值
为贵阳市小学生提供标准化、系统化的在线测评服务，帮助教育部门了解学生学习水平，支持教育决策。

## 2. 核心功能模块

### 2.1 用户系统
- **学生入口**
  - 身份证号登录
  - 学生信息验证
  - 密码找回功能
  
- **教师/管理员入口**
  - 账号密码登录
  - 权限管理
  - 批量导入学生信息

### 2.2 测评系统
根据流程图，测评流程包括：
1. **测评发布** - 教育局发布测评任务
2. **学生报名** - 学生选择参加的测评
3. **资格审核** - 系统自动或人工审核
4. **测评场次分配** - 分配考试时间和批次
5. **测评实施** - 在线答题
6. **自动阅卷** - 客观题自动批改
7. **成绩发布** - 成绩查询和统计
8. **认证申请** - 证书申请和发放

### 2.3 题目类型支持
- **语文**：选择题、填空题、阅读理解
- **数学**：选择题、计算题、应用题
- **科学（计算机）**：选择题、编程题（图形化编程）
- **其他学科**：根据需求扩展

### 2.4 成绩管理
- 成绩查询（学生/家长）
- 成绩统计分析（教师/管理员）
- 成绩报表导出
- 证书生成和打印

## 3. 系统架构设计

### 3.1 三层架构
根据系统架构图：

```
┌─────────────────────────────────────┐
│         前端展示层                    │
│  - 学生端Web界面                      │
│  - 教师管理端                         │
│  - 移动端适配                         │
└─────────────────────────────────────┘
                ↓ API
┌─────────────────────────────────────┐
│         业务逻辑层                    │
│  - 用户认证服务                       │
│  - 测评管理服务                       │
│  - 题库管理服务                       │
│  - 成绩分析服务                       │
└─────────────────────────────────────┘
                ↓
┌─────────────────────────────────────┐
│         数据存储层                    │
│  - 用户数据库                         │
│  - 题库数据库                         │
│  - 成绩数据库                         │
│  - 文件存储（试题图片等）              │
└─────────────────────────────────────┘
```

### 3.2 技术栈建议

**前端**
- React/Vue.js - 主框架
- Ant Design/Element UI - UI组件库
- TypeScript - 类型安全
- Axios - API通信

**后端**
- Node.js + Express 或 Python + Django/FastAPI
- JWT - 身份认证
- Redis - 缓存和会话管理
- WebSocket - 实时通信（考试倒计时等）

**数据库**
- PostgreSQL - 主数据库
- Redis - 缓存层
- MinIO/OSS - 文件存储

**部署**
- Docker - 容器化
- Nginx - 反向代理
- CI/CD - GitLab CI 或 GitHub Actions

## 4. 数据库设计（核心表）

### 4.1 用户相关
```sql
- users (用户表)
  - id, username, password, role, real_name, id_card
  - school_id, grade, class, created_at

- schools (学校表)
  - id, name, district, address, contact

- teachers (教师表)
  - id, user_id, teacher_no, subjects
```

### 4.2 测评相关
```sql
- exams (测评表)
  - id, title, subject, grade, start_time, end_time
  - duration, total_score, status

- questions (题目表)
  - id, exam_id, type, content, options, answer
  - score, order, difficulty

- exam_sessions (考试场次)
  - id, exam_id, session_no, start_time, capacity

- student_exams (学生考试记录)
  - id, student_id, exam_id, session_id, status
  - start_time, submit_time, score
```

### 4.3 成绩相关
```sql
- answers (答题记录)
  - id, student_exam_id, question_id, answer
  - score, is_correct

- certificates (证书表)
  - id, student_id, exam_id, cert_no
  - issue_date, level
```

3. 自动评分（客观题）
4. 成绩查询
5. 基础题库管理

### P1 - 应该实现（第二阶段）

2. 批量导入学生/题目
3. 考试场次管理
4. 成绩统计分析
5. 证书生成

### P2 - 可以实现（第三阶段）

## 6. UI/UX设计要点

### 6.1 设计原则
- **简洁友好**：适合小学生使用，大字体、清晰图标
- **色彩活泼**：使用明亮、温暖的色彩方案
- **交互简单**：减少复杂操作，引导清晰
- **响应式设计**：支持平板、电脑多终端

### 6.2 关键页面
1. **登录页**：参考中国电子学会风格，蓝色主题
2. **考试大厅**：显示可参加的测评列表
3. **答题界面**：清晰的题目展示，进度条，倒计时
4. **成绩查询**：图表化展示成绩分析
5. **管理后台**：数据统计仪表盘

## 7. 安全与合规

### 7.1 数据安全
- HTTPS加密传输
- 敏感数据加密存储
- 定期数据备份
- 访问权限控制

### 7.2 合规要求
- 遵守《个人信息保护法》

- 考试公平性保障

## 10. 风险与对策

| 风险 | 可能性 | 影响 | 对策 |
|-----|-------|------|------|
| 并发压力 | 高 | 高 | 负载均衡，分批次考试 |
| 网络中断 | 中 | 高 | 本地缓存，断点续考 |
| 作弊行为 | 中 | 中 | 随机题序，时间限制 |
| 数据泄露 | 低 | 高 | 加密存储，权限控制 |
