## 三、算法规则 — 成熟方案详解（重点）

> 本轮需实现：算法①②③（含 pgvector，已确认允许安装）。
> 算法④⑤（AI）本轮暂缓，保留方案备后续。

### 算法① 题目同质化检测（覆盖 C3，A3-同质后续接）

**漏斗式四层**：

| 层级 | 方法 | 作用 |
|------|------|------|
| L0 结构化硬规则 | 题型+科目+年级+知识点集合+难度 全等 → 标记疑似 | 快速粗筛 |
| L1 文本指纹 | 正文+选项拼接 → `jieba` 分词 → 去停用词 → **SimHash**（64bit），汉明距离 ≤3 为候选；海量场景 **MinHash + LSH** | 海量近邻粗筛 |
| L2 语义精排 | 中文句向量（`BAAI/bge-small-zh`）编码 → 余弦相似度（pgvector） | 识别"换皮"题 |
| L3 综合评分 | `score = w1·结构相似 + w2·文本相似 + w3·语义相似` | 决策 |

**确认阈值**：综合分 ≥ **0.85** 标记同质；≥ **0.95** 强提示。

**落地**：入库/发布时异步算指纹与向量，存 `question_fingerprint` + `question_embedding`（pgvector）；组卷 `addQuestionToActivity` 时对当前已选集查重，超阈值弹窗。先上 L0+L1（无外部依赖），量大再接 L2。

### 算法② 碎片化学习推荐（D1）

```
Score(q) = α · masteryFactor(q)        // 薄弱知识点优先（掌握度低分高）
         + β · zpdFactor(q)             // 难度匹配：最近发展区（约 60-75% 正确率）
         + γ · spacedRepetition(q)      // 错题间隔重复：艾宾浩斯 + SM-2
         + δ · noveltyFactor(q)         // 近期未做过
         - ε · homogenizePenalty(q)     // 与本批次已推题同质降权（复用算法①）
```

- 掌握度：复用 `student_knowledge_stats`；ZPD：题目难度 vs 学生能力；SM-2：错题复习调度。
- **确认**：权重统一，不按年级区分。
- 冷启动：按年级/科目/教材知识点顺序 + 中等难度随机。
- 落地：`backend/src/services/recommend/QuestionRecommender.js`。

### 算法③ 每日推题（D3）

**每日 N=10 题构成**：

| 槽位 | 占比 | 来源 | 算法 |
|------|------|------|------|
| 错题复习 | ~30-40% | 错题集 | SM-2 到期优先 |
| 弱项新题 | ~40% | 掌握度 < 阈值的知识点 | 算法② |
| 难度匹配新题 | 剩余 | ZPD 区间新题 | 算法② |

约束：知识点轮换、按选科平衡、当日/近 N 天不重（批次内去重复用算法①）；连胜累计答对触发积分/成就。结果缓存 `daily_question_sets`。**无现成任务队列** → 新建 node-cron 每日生成 + 首次打开即时补算。

### 【本次暂缓】算法④ AI 预审（A3）/ 算法⑤ 问答题 AI 判分（A6）

保留原方案（LLM + 结构化 JSON / rubric 评分 + 低置信度转人工 + 仅测评启用），待后续 AI 专项立项。问答题本轮仍走人工评卷（`grading.js`）。

## 四、各功能实施要点

### A. 题库与审核

| 项 | 实施要点 | 主要文件 |
|----|----------|----------|
| A1 提级（含原A4） | ① `POST /question-bank/:id/promote`（区级所有者发起，创建 `practice_municipal` 记录，状态 `pending_review`，待市级审核）；② `POST /question-bank/:id/admin-promote`（市级管理员主动提级本市下辖任意区级题，直接生效）。提级均指向同一 `draft_id` | `questionBank.js` / `questionReview.js` / `QuestionBank.js` |
| A2 隐藏 | 启用 `is_hidden`：管理端开关；题库浏览/自由练习查询过滤；**可见性规则见下方确认** | `QuestionBank.js`、前端开关 |
| A5 使用统计 | 答题提交时 `usage_count++`，正确率 = 正确人数/提交人数；超阈值告警（使用次数>X 或 正确率<Y 建议换题） | `studentActivities.js`、`QuestionBank.js` |

**A2 可见性规则（已确认）**：
- `is_hidden=true` 的题目，对**出题人、审核人、各级上级管理员**始终可见；
- 测评**开始时间之前**：对其他老师和学生**不可见**（题库浏览/自由练习均不可见）；
- **测评模式**中：必须在该测评**开始时间之后**才对学生出现（考试时段正常作答）；
- **解除隐藏**：由上级管理员**手动**操作（不自动解除）。

### B. 积分系统

| 项 | 实施要点 | 主要文件 |
|----|----------|----------|
| B1 刷题积分 | 答题正确后发放；难度系数 + 单难度上限 + 收益递减（参数见下）；新建 `PointsPolicy` 服务 | `studentActivities.js`、`StudentPoints.js`、新增 `PointsPolicy.js` |
| B2 成就积分 | 补齐刷题类成就模板（数量/连胜/科目） | `achievement/templates` |

**B1 积分参数（已采纳默认值，可调）**：

| 难度 | 基础分 | 单难度每日积分上限 |
|------|--------|-------------------|
| 简单 easy | 1 | 20 |
| 中等 medium | 3 | 50 |
| 困难 hard | 6 | 80 |

```
收益递减：第k题积分 = base[difficulty] × 1/(1 + 0.15 × k)，k=当日该难度已答对题数
每日总积分上限：150 分（三难度合并封顶，防刷）
连胜奖励：每连续答对 5 题 +5 分（中断归零）
错题重做答对：基础分 × 0.5，每题最多给 2 次重做积分
```

- 单个学生对同一题**只能纠错 1 次**（合理默认，防恶意刷）。

## 六、最终决策记录（已逐条确认）

### 产品与流程
1. **A1 提级**：区级推市级**需市级管理员审核**；市级管理员主动提级**本市范围内无限制**（可提本市下辖所有区级题）。
2. **A4**：并入 A1 —— 市级管理员主动提级即通过，不另做"审核通过自动提级"。
3. **A2 隐藏可见性**：对出题/审核老师 + 各级上级管理员可见；测评开始前对其他老师和学生不可见；**测评模式中须在测评开始时间之后才对学生出现**；测评结束后由上级管理员手动解除。
4. **C5 纠错**：主要由出题人和审核人处理；**累计 3 次提醒上一级（区级）**；**累计 5 次封顶**；暂无责任界定。
5. **C4 配额**：草稿+已发布总数；超限阻止创建+警告；老师可向上级申请追加；**额度分配权给市级管理员**。

### 积分
6. **B1 参数**：授权提案 → 已采纳（见第四节积分参数表）。
7. **E2 消费**：先做皮肤/头像框/名字颜色 3 类，后续可扩展；**名字颜色不影响他人查看**。

### 算法阈值
8. **算法①**：0.85 标记同质，0.95 强提示。
9. **算法②**：权重统一，不按年级调整。

### 端与角色
10. **D2/E4**：小程序后置，**本轮先做响应式 Web**。
11. **E4 家长**：只读看孩子大部分数据+练习记录，不能修改；**可代孩子报名测评**；**一个孩子只关联一个家长**。
12. **C2 虚拟练习**：与真实题库关联。

### 工程依赖
13. **pgvector**：允许安装。
14. **定时任务**：无现成基础设施 → 新建 node-cron。
15. **AI（A3/A6）**：本轮暂缓，后续专项立项。

### 本轮新增确认
16. **A2 测评可见性**：隐藏题目在测评模式中须测评开始时间之后才对学生出现。
17. **C5 计数维度**：累计计数（题目维度），5 次封顶；单学生同一题 1 次（合理默认）。
18. **D4 入库时机**：答错即自动进入错题集（判题型在判题完成后入库）。

## 八、阶段一（P0）可执行 Task 清单

> 阶段一目标：闭环"做题→积分→错题"，堵住数据泄露与积分缺口。
> 顺序：数据库迁移先行 → 各功能可并行 → 统一联调。

### Task 1.0 数据库迁移（前置）
- 新增迁移 `database/migrations/043_phase1_features.sql`：
  - `question_promotions`（提级申请：draft_id, from_scope, to_scope, requested_by, status, reviewed_by, reviewed_at）
  - `points_policy`（key/value 配置：基础分、上限、λ、总上限）
  - `student_points_daily`（student_id, date, difficulty, answered_correctly_count, earned_points）
  - `student_wrong_questions`（student_id, draft_id/question_id, subject, knowledge_points, error_count, last_wrong_at, status, source_activity_id）
  - 同步更新 `schema.sql`。
- **验收**：迁移在 Docker PG 上执行成功，表结构符合预期。

### Task 1.1 A2 隐藏题库（独立，快）
- 后端 `QuestionBank.js`：题库浏览/自由练习查询加 `is_hidden` 过滤（非授权角色不可见）。
- 学生端题目查询：`is_hidden=true` 且未到关联测评开始时间的，过滤掉。
- 测评答题查询：允许 `is_hidden` 题目在测评开始后可见（按活动 start_time 判断）。
- 前端：管理端题库列表加"隐藏"开关列。
- **验收**：隐藏题在题库浏览/练习对学生不可见；测评开始后正常出现；管理员可手动解除。

### Task 1.2 A1 提级（独立）
- 后端 `POST /question-bank/:id/promote`：区级所有者发起 → 建 `practice_municipal` 记录 + `question_promotions` 审核（pending）。
- 后端 `POST /question-bank/:id/admin-promote`：市级管理员主动提级本市下辖区级题 → 直接生效（status=published，需审核流程则走 review）。
- 前端：区级题库列表加"申请提级"按钮；市级管理员待审列表加"通过/拒绝"。
- **验收**：区级发起后市级可见待审；市级通过后题目在市级题库可见；市级主动提级立即生效。

### Task 1.3 C1 发布置灰（独立，快）
- 后端 `paperGenerationService`：`addQuestionToActivity`/`removeQuestionFromActivity`/`clearActivityPaper`/`updateActivityQuestion`/`reorderQuestions` 校验活动 `status=published` → 返回 403 中文错误。
- 前端组卷页：活动已发布时禁用相关按钮、显示"已发布，不可修改"提示。
- **验收**：发布后添加/移除/清空接口返回 403；前端按钮置灰。

### Task 1.4 B1 刷题积分（较重，单独推进）
- 新增 `services/points/PointsPolicy.js`：读取 `points_policy` 配置，实现难度系数、收益递减（λ=0.15）、单难度上限、每日总上限、连胜、错题重做规则。
- 改 `studentActivities.js` 答题提交流程：判题结果为正确时，按策略计算并 `StudentPoints.addPoints`；记录到 `student_points_daily`。
- 错题重做入口同样走策略（×0.5，每题≤2 次）。
- 前端：积分变动提示、每日上限到达提示。
- **验收**：答对按难度发分并衰减；单难度/总上限触发后 0 分但仍计做题数；连胜断则归零；积分交易有记录。

### Task 1.5 D4 错题集（独立）
- 新增 `models/WrongQuestion.js` + `routes/wrongQuestions.js`：
  - 入库：答题判错时自动写入（编程题等异步判题完成后写入）。
  - 列表：按科目/知识点筛选、分页。
  - 重做、移除、标记已掌握。
- 前端：学生端"错题集"页面（响应式）。
- **验收**：答错自动入库；编程题判错后入库；可筛选/重做/移除。

### Task 1.6 阶段一联调与测试
- 后端：Jest 单测覆盖 PointsPolicy、提级、隐藏过滤、错题入库。
- 前端：关键流程自测。
- 文档：更新 `docs/DEVELOPMENT_STATUS.md`。
