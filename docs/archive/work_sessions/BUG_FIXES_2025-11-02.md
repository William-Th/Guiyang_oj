### 问题 1: 科目列表未从配置读取

- 授权管理页面中，授权科目也是硬编码的

3. 在 `PermissionManagement.tsx` 中导入 `SUBJECTS`
4. 将授权科目选项改为使用 `SUBJECTS` 配置

### 问题 2: 用户管理中角色权限限制缺失

**问题描述**:
- 学生角色可以被更改
- 低权限管理员可以编辑高权限管理员
- 例如：校级管理员可以设置区级管理员或市级管理员

**影响范围**:
- `frontend/src/pages/admin/UserManagement.tsx`

**修复方案**:
1. 定义角色层级体系（ROLE_HIERARCHY）
   - student: 1
   - teacher: 2
   - school_admin: 3
   - municipal_school_admin / base_school_admin: 4
   - district_admin: 5
   - municipal_admin: 6
   - system_admin: 7

2. 创建 `getManageableRoles()` 函数
   - 只返回低于当前用户权限的角色

3. 修改 `handleEditUser()` 函数
   - 添加权限检查
   - 禁止编辑高于或等于自己权限的用户

4. 修改角色选择下拉框
   - 学生角色不可更改
   - 只显示可管理的角色选项

**修改文件**:
```typescript
// frontend/src/pages/admin/UserManagement.tsx

// 角色层级定义
const ROLE_HIERARCHY: Record<UserRole, number> = {
  'student': 1,
  'teacher': 2,
  'school_admin': 3,
  'municipal_school_admin': 4,
  'base_school_admin': 4,
  'district_admin': 5,
  'municipal_admin': 6,
  'system_admin': 7,
};

// 获取可管理的角色列表
const getManageableRoles = (currentUserRole: UserRole) => {
  const currentLevel = ROLE_HIERARCHY[currentUserRole];
  return ROLE_OPTIONS.filter(option => option.level < currentLevel);
};

// 编辑用户时检查权限
const handleEditUser = (user: User) => {
  const currentUserLevel = ROLE_HIERARCHY[currentUser?.role as UserRole] || 0;
  const targetUserLevel = ROLE_HIERARCHY[user.role];

  if (targetUserLevel >= currentUserLevel) {
    message.warning('您无权编辑该用户，该用户权限高于或等于您的权限');
    return;
  }
  // ...
};

// 角色选择
<Select disabled={editingUser?.role === 'student'}>
  {getManageableRoles(currentUser?.role as UserRole).map(roleOption => (
    <Option key={roleOption.value} value={roleOption.value}>
      {roleOption.label}
    </Option>
  ))}
</Select>
```

**验证结果**:
- ✅ 学生角色不可更改（选择框禁用）
- ✅ 校级管理员只能管理教师和学生
- ✅ 区级管理员可以管理校级管理员、教师和学生
- ✅ 尝试编辑高权限用户时显示警告信息

### 问题 3: 题库管理中年级选项未根据科目联动

**问题描述**:
- 题库管理筛选中，年级选项是固定的（一年级-高三）
- 没有根据选择的科目动态调整年级选项
- 例如：信息科技只支持三年级-九年级，但仍显示所有年级

**影响范围**:
- `frontend/src/pages/teacher/QuestionBankPage.tsx`
- `frontend/src/pages/teacher/QuestionFormPage.tsx`（已在问题1中修复）

**修复方案**:
1. 在 `QuestionBankPage.tsx` 中导入 `getGradesBySubject` 和 `getAllGrades`
2. 修改年级选择下拉框
   - 根据选择的科目动态显示对应的年级选项
   - 未选择科目时禁用年级选择或显示所有年级
3. 切换科目时清空年级选择

**修改文件**:
```typescript
// frontend/src/pages/teacher/QuestionBankPage.tsx
import { SUBJECTS, getGradesBySubject, getAllGrades } from '../../config/subjects';

// 科目选择（清空年级）
<Select
  placeholder="选择科目"
  onChange={(value) => {
    setFilters({ ...filters, subject: value, grade: undefined });
  }}
  value={filters.subject}
>
  {SUBJECTS.map(subject => (
    <Select.Option key={subject.value} value={subject.value}>
      {subject.label}
    </Select.Option>
  ))}
</Select>

// 年级选择（根据科目联动）
<Select
  placeholder={filters.subject ? "选择年级" : "请先选择科目"}
  disabled={!filters.subject}
  value={filters.grade}
  onChange={(value) => setFilters({ ...filters, grade: value })}
>
  {filters.subject
    ? getGradesBySubject(filters.subject).map(grade => (
        <Select.Option key={grade.value} value={grade.value}>
          {grade.label}
        </Select.Option>
      ))
    : getAllGrades().map(grade => (
        <Select.Option key={grade.value} value={grade.value}>
          {grade.label}
        </Select.Option>
      ))
  }
</Select>
```

**验证结果**:
- ✅ 选择"数学"时，年级显示"一年级-九年级"
- ✅ 选择"信息科技"时，年级显示"三年级-九年级"
- ✅ 切换科目时，年级选择自动清空
- ✅ 未选择科目时，年级选择被禁用

## 📝 注意事项

1. **配置文件依赖**: 所有科目和年级配置现在依赖 `frontend/src/config/subjects.ts`

   - 修改后需重新构建前端Docker镜像

3. **向后兼容**: 修改后的系统与现有数据库完全兼容
