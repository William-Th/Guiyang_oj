# 个人资料页面手动测试指南

## 功能概述

本次开发实现了个人资料页面的角色特定信息显示功能。系统会根据用户角色（教师、学生、管理员）显示不同的个人信息字段。

## 开发完成内容

### 1. 数据库设计 ✅
- 验证现有表结构支持所有需求字段
- 无需修改数据库schema

### 2. 后端API开发 ✅
- **文件**: `backend/src/models/User.js`
- **新增方法**: `getDetailedProfile(userId)` (Lines 429-535)
- **功能**: 根据用户角色返回详细信息
  - 教师: teacherNo, subjects[], title, school, district
  - 学生: studentNo, grade, class, school, district
  - 管理员: managementLevel, school, district, permissionScope

- **文件**: `backend/src/routes/users.js`
- **修改**: 更新 `/profile` 路由使用新方法 (Lines 8-21)

### 3. 前端开发 ✅
- **文件**: `frontend/src/store/authSlice.ts`
  - 扩展User interface，添加角色特定字段

- **文件**: `frontend/src/pages/ProfilePage.tsx`
  - 更新页面显示逻辑，根据角色显示不同字段
  - 教师显示: 教师编号、任教科目、职称、学校、区域
  - 学生显示: 学号、年级、班级、学校、区域
  - 管理员显示: 管理级别、管理学校/区域

- **文件**: `frontend/src/pages/LoginPage.tsx`
  - 更新登录流程，先获取token，再获取详细profile
  - 确保Redux store存储完整的用户信息

### 4. Docker重建 ✅
- 后端服务重建完成
- 前端服务重建完成
- Nginx代理重启完成

## 已知问题和注意事项

### ⚠️ 代理问题
- 如果使用curl或浏览器访问时遇到503错误
- **解决方案**: 确保禁用本地代理，或配置代理排除localhost
- **测试命令**: `curl --noproxy "*" http://localhost`

### ⚠️ API测试503错误
- API测试脚本 `tests/api/profile-api-test.js` 遇到503错误
- **原因**: 测试脚本可能受到本地代理影响
