### 1.7 测评活动管理

**路径**：`/admin/assessments`

![测评活动管理](screenshots/12_admin_assessments.png)

**活动类型**：

| 类型 | 标识 | 说明 |
|------|------|------|
| 测评 | `assessment` | 正式考试，有严格时间限制 |
| 练习 | `practice` | 自主练习，可配置重做 |

**创建活动流程**：

1. 选择类型（测评/练习）和科目
2. 选择年级（一年级 ~ 高三）和能力等级（L1 ~ L7）
3. 设定时间范围和时长限制（限时/不限时）
4. 设置及格分数和总分
5. 选择发布范围（市级/区级/校级）
6. 配置是否允许重做及最大尝试次数
7. 提交后进入组卷环节

**活动状态流转**：

```
草稿 → 已发布 → 进行中 → 已结束
```

**组卷功能**（`/admin/assessments/:id/paper`）：

- 从题库按条件筛选题目（题型、难度、科目）
- 添加到试卷并设置分值
- 支持手动选题和自动抽题
- 预览试卷总分和时间配置

## 六、系统架构与技术亮点

### 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | React 18 + TypeScript + Ant Design 5 + Redux Toolkit + Vite |
| 后端 | Node.js + Express.js + PostgreSQL 15 + Redis |
| 判题 | 独立 judge-service 服务，沙箱执行代码 |
| 部署 | Docker Compose + Nginx 反向代理 |
| 图表 | Recharts（柱状图、雷达图） |
| 测试 | Playwright (E2E) + Jest (单元/API) |

### 服务架构

```
浏览器 → Nginx(:8080) → Frontend(:3100)
                        → Backend(:3003) → PostgreSQL(:5432)
                                          → Redis(:6379)
                        → Judge Service(:3002)
```

### 核心亮点

| 亮点 | 说明 |
|------|------|
| 多级权限体系 | 5 层角色 × 6 个行政区，数据严格隔离 |
| 7 种题型全覆盖 | 单选/多选/判断/填空/编程/匹配/问答 |
| 编程题自动判题 | 独立判题服务，沙箱执行，多测试用例 |
| 题库质量管控 | 草稿→提交→审核→发布，完整工作流 |
| 成就积分体系 | 5 级稀有度、4 大类别、积分激励、游戏化学习 |
| 能力等级评估 | 数学和信息技术各 7 个等级（L1~L7），雷达图可视化 |
| 数据可视化 | 柱状图、雷达图、进度条、统计卡片 |
| 答题安全 | 倒计时、双保存（服务端+localStorage）、自动交卷 |
| 证书系统 | 自动生成 + 在线验证真伪 |
| 一键部署 | Docker Compose 编排全部服务 |
