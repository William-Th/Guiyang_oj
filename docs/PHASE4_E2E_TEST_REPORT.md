## 🔧 修复的问题

### 修复1: 选择器文本错误
**问题**: 测试使用错误的标签文本
**修复**:
- ❌ `label:has-text("选择用户")` → ✅ `label:has-text("选择教师")`
- ❌ `label:has-text("适用科目")` → ✅ `label:has-text("授权科目")`

### 修复2: 按钮文本错误
**问题**: 按钮文本不匹配
**修复**:
- ❌ `button:has-text(/创建题目/)` → ✅ `button:has-text(/新建题目/)`

### 修复3: 控件类型错误
**问题**: 测试将 Select 下拉框当作 Radio 按钮处理
**修复**:
```typescript
// ❌ 错误 - 使用 Radio
const singleChoiceOption = page.getByRole('radio', { name: /单选题/ });
await singleChoiceOption.click();

// ✅ 正确 - 使用 Select
await page.locator('label:has-text("题型")').locator('..').locator('.ant-select').click();
await page.waitForTimeout(500);
const singleChoiceOption = page.getByRole('option', { name: /单选题/ });
await singleChoiceOption.evaluate((el: HTMLElement) => el.click());
```

### 修复4: Placeholder 选择器语法错误
**问题**: 使用了无效的 `placeholder=` 选择器语法
**修复**:
- ❌ `page.locator('placeholder=选择题库范围')`
- ✅ `page.getByPlaceholder('选择题库范围')`

### 修复5: 严格模式违规
**问题**: 定位器找到多个匹配元素
**修复**:
```typescript
// ❌ 错误 - 找到3个匹配元素
await expect(page.locator('text=审核工作台').or(page.locator('text=待审核题目'))).toBeAttached();

// ✅ 正确 - 使用 .first() 选择第一个
await expect(page.locator('text=审核工作台').or(page.locator('text=待审核题目')).first()).toBeAttached();
```

### 修复6: 用户角色错误
**问题**: E2E-004 使用管理员登录，但审核工作台仅在教师菜单中
**修复**:
- ❌ `await loginAsAdmin(page);`
- ✅ `await loginAsTeacher(page);` + 注释说明原因

## 💡 问题根因分析

### 1. 表单加载问题
**影响测试**: E2E-001, E2E-002, E2E-003
**症状**: 点击按钮后无法找到表单元素
**可能原因**:
- 异步加载延迟超出超时时间
- 模态框/页面渲染逻辑变更
- 表单元素 DOM 结构与选择器不匹配
- 页面可能显示错误或空状态

### 2. 选择器维护问题
**影响**: 所有测试
**症状**: 多次出现选择器文本/类型错误
**根本原因**:
- E2E 测试编写时未参考实际前端代码
- 前端组件更新后测试未同步维护
- 缺少自动化验证机制
