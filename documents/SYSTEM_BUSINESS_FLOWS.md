## 系统概览

贵阳市小学生测评平台是一个面向小学教育的在线测评服务系统，支持多层级管理、题库协作和活动组织。系统围绕三大核心业务体系构建：

| 业务体系 | 核心功能 | 主要用户 |
|---------|---------|---------|
| **账号与权限系统** | 用户身份管理、多层级权限控制、区域/学校管理 | 系统管理员、区域管理员、学校管理员 |
| **题库系统** | 题目创建、审核流程、分类管理、协作编辑 | 教师、学科专家、审核员 |
| **活动中心系统** | 练习/测评创建、学生参与、成绩统计、证书生成 | 教师、学生、管理员 |

### 系统用户角色

| 角色 | 英文标识 | 权限范围 | 主要职责 |
|------|---------|---------|---------|
| **学生** | student | 个人活动 | 参与练习/测评、查看成绩、下载证书 |
| **教师** | teacher | 班级/年级 | 创建题目、组织练习、查看学生成绩 |
| **学校管理员** | school_admin | 本校范围 | 管理本校教师、审核题目、组织校级测评 |
| **区域管理员** | district_admin | 区域范围 | 管理区域学校、协调教学资源 |
| **市级管理员** | municipal_admin | 全市范围 | 管理全市数据、发布市级测评 |
| **系统管理员** | system_admin | 全系统 | 系统配置、用户管理、数据维护 |

## 账号与权限系统

### 系统概述

账号与权限系统是平台的基础设施，负责用户身份验证、权限控制和组织结构管理。系统采用多层级权限模型，支持从市级到学校的层级管理。

### 核心业务流程

#### 1. 用户注册与登录流程

```
┌─────────────┐
│  访问平台   │
└──────┬──────┘
       │
       ▼
┌─────────────┐      是
│  已有账号？ ├──────────────────┐
└──────┬──────┘                  │
       │否                       │
       ▼                         ▼
┌─────────────┐          ┌─────────────┐
│  学生注册   │          │   用户登录   │
│ (身份证号)  │          │ (账号/密码)  │
└──────┬──────┘          └──────┬──────┘
       │                        │
       ▼                        ▼
┌─────────────┐          ┌─────────────┐
│ 教师/管理员 │          │  JWT认证    │
│  账号申请   │          │  生成Token  │
└──────┬──────┘          └──────┬──────┘
       │                        │
       ▼                        ▼
┌─────────────┐          ┌─────────────┐
│ 管理员审核  │          │  进入系统   │

```

**关键特性**:
- **学生**: 使用18位身份证号登录（出生日期验证）
- **教师/管理员**: 使用用户名+密码登录
- **会话管理**: JWT Token有效期24小时，支持自动续期
- **安全机制**: bcrypt密码加密、登录失败限制、会话绑定

### 核心功能模块

#### 3.1 用户管理

**功能描述**: 管理平台所有用户账号，支持批量导入、信息修改、账号禁用等操作。

**业务流程**:

1. **批量导入学生**:
   - 管理员上传Excel文件（学生名单）
   - 系统验证数据格式（姓名、身份证号、年级、班级）
   - 自动生成账号（身份证号）
   - 批量创建用户记录
   - 导出导入结果报告

2. **教师账号申请**:
   - 教师填写注册表单（姓名、联系方式、学校、科目）
   - 提交至学校管理员审核
   - 管理员核实教师身份
   - 批准后激活账号
   - 教师收到账号信息

3. **账号状态管理**:
   - **激活**: 新账号默认激活状态
   - **禁用**: 暂时冻结账号（违规/离职）
   - **删除**: 永久删除账号及关联数据
   - **重置密码**: 管理员强制重置密码

**权限控制**:
- 系统管理员: 管理所有用户
- 区域/学校管理员: 管理所属范围内用户
- 教师: 无用户管理权限

#### 3.2 组织架构管理

**功能描述**: 管理市-区-学校-年级-班级的层级结构，支持组织调整和数据迁移。

**层级结构**:

```
贵阳市教育局
├── 云岩区教育局
│   ├── 省府路小学
│   │   ├── 一年级
│   │   │   ├── 一年级1班
│   │   │   └── 一年级2班
│   │   └── 二年级
│   └── 实验小学
└── 南明区教育局
    └── 甲秀小学
```

**管理操作**:

1. **创建学校**: 系统管理员/区域管理员创建新学校
2. **分配管理员**: 指定学校管理员
3. **调整结构**: 学校转区、班级调整
4. **数据迁移**: 学生转学、教师调动时自动迁移数据

#### 3.3 权限审计日志

**功能描述**: 记录所有权限相关操作，支持审计追溯和安全分析。

**记录内容**:
- 登录/登出记录（IP地址、设备信息）
- 权限变更记录（授予/撤销操作）
- 敏感操作记录（数据删除、批量修改）
- 异常行为记录（登录失败、权限越界）

**审计报表**:
- 用户活跃度统计
- 权限分配情况
- 安全事件汇总
- 异常行为分析

## 题库系统

### 系统概述

题库系统是平台的内容核心，支持多种题型创建、协作审核和分类管理。系统采用草稿-审核-发布的三阶段流程，确保题目质量。

### 核心业务流程

#### 1. 题目创建与审核流程

```
┌─────────────┐
│ 教师创建题目│
└──────┬──────┘
       │
       ▼
┌─────────────┐      保存
│  填写题目内容├──────────►┌─────────────┐
│  - 题干     │            │  草稿箱     │
│  - 选项     │            │  (可编辑)   │
│  - 答案     │            └─────────────┘
│  - 解析     │
└──────┬──────┘
       │提交审核
       ▼
┌─────────────┐
│  待审核列表 │
│  (锁定编辑) │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 审核员审核  │
│  - 内容正确性│
│  - 难度合理性│
│  - 格式规范性│
└──────┬──────┘
       │
       ├──通过──────►┌─────────────┐
       │             │  已发布     │
       │             │  (可用)     │
       │             └─────────────┘
       │
       └──拒绝──────►┌─────────────┐
                     │  退回修改   │
                     │  (回到草稿) │
                     └─────────────┘
```

**题目状态流转**:

| 状态 | 说明 | 可执行操作 |
|-----|------|-----------|
| **draft** | 草稿 | 编辑、删除、提交审核 |
| **pending_review** | 待审核 | 撤回、审核员审核 |
| **approved** | 已发布 | 查看、引用到活动 |
| **rejected** | 审核拒绝 | 修改后重新提交 |

- **质量评价**: 使用后统计错误率、区分度等指标

### 核心功能模块

#### 2.1 题目创建（支持5种题型）

**1. 单选题 (Single Choice)**

**题目结构**:
```json
{
  "type": "single_choice",
  "content": "题干文字",
  "options": [
    {"label": "A", "text": "选项A内容"},
    {"label": "B", "text": "选项B内容"},
    {"label": "C", "text": "选项C内容"},
    {"label": "D", "text": "选项D内容"}
  ],
  "correct_answer": "A",
  "explanation": "解析文字",
  "score": 5
}
```

**创建流程**:
1. 选择题型: 单选题
2. 输入题干（支持富文本、图片）
3. 添加选项（2-6个，默认4个）
4. 标记正确答案
5. 填写解析说明
6. 设置分值
7. 保存到草稿箱

**验证规则**:
- 题干不能为空
- 至少2个选项
- 必须选择1个正确答案
- 选项内容不能重复

**2. 多选题 (Multiple Choice)**

**题目结构**:
```json
{
  "type": "multiple_choice",
  "content": "题干文字",
  "options": [
    {"label": "A", "text": "选项A内容"},
    {"label": "B", "text": "选项B内容"},
    {"label": "C", "text": "选项C内容"},
    {"label": "D", "text": "选项D内容"}
  ],
  "correct_answer": ["A", "C"],
  "explanation": "解析文字",
  "score": 10
}
```

**验证规则**:
- 至少选择2个正确答案
- 正确答案不能等于选项总数（否则应为判断题）

**3. 判断题 (True/False)**

**题目结构**:
```json
{
  "type": "true_false",
  "content": "判断题题干",
  "correct_answer": "true",  // 或 "false"
  "explanation": "解析文字",
  "score": 3
}
```

**4. 填空题 (Fill in Blank)**

**题目结构**:
```json
{
  "type": "fill_blank",
  "content": "天空是___色的，草地是___色的。",
  "blanks": [
    {
      "position": 1,
      "correct_answers": ["蓝", "湛蓝", "碧蓝"],
      "case_sensitive": false
    },
    {
      "position": 2,
      "correct_answers": ["绿", "翠绿"],
      "case_sensitive": false
    }
  ],
  "explanation": "解析文字",
  "score": 8
}
```

**特性**:
- 支持多空填空
- 支持多个正确答案（同义词）
- 支持大小写敏感/不敏感
- 自动判分

**5. 简答题 (Subjective)**

**题目结构**:
```json
{
  "type": "subjective",
  "content": "请描述春天的景色特点。",
  "reference_answer": "参考答案文字",
  "scoring_criteria": [
    "描述了至少3个景物特征",
    "使用了恰当的修辞手法",
    "语句通顺，逻辑清晰"
  ],
  "score": 15
}
```

**特性**:
- 需要人工评分
- 提供参考答案和评分标准
- 支持批量阅卷

#### 2.2 题目审核

**审核员角色**:
- 学校管理员
- 学科专家（特殊授权）
- 区域/市级管理员

**审核流程**:

```
┌─────────────────┐
│ 审核员查看      │
│ 待审核列表      │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 逐题审核        │
│ - 查看题目内容  │
│ - 检查答案正确性│
│ - 评估难度等级  │
└────────┬────────┘
         │
         ▼
      ┌──┴──┐
   批准│    │拒绝
      ▼    ▼
┌──────────┐  ┌──────────┐
│ 发布题目 │  │ 填写拒绝 │
│          │  │ 理由     │
└──────────┘  └────┬─────┘
                   │
                   ▼
             ┌──────────┐
             │ 通知教师 │
             │ 修改题目 │
             └──────────┘
```

**审核维度**:

| 维度 | 检查项 | 不通过情况 |
|-----|--------|-----------|
| **内容正确性** | 题干表述、答案准确性 | 答案错误、题干歧义 |
| **难度合理性** | 是否符合年级水平 | 超纲、过于简单 |
| **格式规范性** | 标点、格式、图片质量 | 格式混乱、图片模糊 |
| **知识点匹配** | 分类是否准确 | 科目/年级选择错误 |
| **选项设计** | 干扰项合理性 | 干扰项明显错误 |

#### 2.4 题目统计分析

**统计指标**:

| 指标 | 计算方法 | 应用场景 |
|-----|---------|---------|
| **使用次数** | 题目被引用到活动的次数 | 评估题目受欢迎程度 |
| **平均分** | 所有作答的平均得分 | 评估题目难度 |
| **错误率** | 答错人数 / 总作答人数 | 识别难题、易错题 |
| **区分度** | 高分组与低分组答对率差异 | 评估题目质量 |
| **作答时长** | 平均作答时间 | 评估题目复杂度 |

**分析报表**:
- 题目使用排行榜
- 知识点覆盖度分析
- 题型分布统计
- 难度分布曲线

### B. 业务规则总结

**题库系统规则**:
1. 草稿状态题目可编辑，提交审核后锁定
2. 审核拒绝后退回草稿状态
3. 已发布题目不可删除（只能归档）
4. 题目编号自动生成，不可修改

**活动系统规则**:
1. 练习活动可重复参与，测评活动只能参与一次
2. 活动发布后不可修改题目（可修改时间和描述）
3. 学生提交后不可再次修改答案
4. 超时自动提交，不影响已答题目得分
5. 及格线不能低于总分的60%

**权限系统规则**:
1. 下级无法管理上级账号
2. 权限变更需要审核
3. 学生账号不可登录管理后台
4. Session过期需要重新登录
